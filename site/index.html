Install

Register this repo as a Claude Code plugin marketplace, then install the plugin. Both commands are slash commands — type them inside Claude Code.

/plugin marketplace add juxstin1/archmap-plugin
/plugin install archmap@archmap

Local development

git clone https://github.com/juxstin1/archmap-plugin.git
cd archmap-plugin
claude --plugin-dir "$(pwd)"

Requirements

  • Claude Code
  • bash (git-bash or WSL on Windows)
  • jq recommended for full .archmap.json parsing

What you get

Interactive canvas

Pan, zoom, click-to-inspect. Dependency edges with curved arrows. Complexity bars per module. Four theme presets.

Self-contained

One HTML file. No CDN. No build step. No runtime dependencies. Works offline, ships via email, lives in your repo.

Drift detection

SessionStart and PostToolUse hooks notice when your code outgrows the map and nudge you to repair — only what changed.

Surgical repair

/archmap:repair patches only stale modules, broken edges, and layout issues. User notes are preserved.

Focus mode

/archmap:focus <module> re-explores a single module in depth — updated types, functions, edges — without regenerating the whole map.

Read-only diff

/archmap:diff reports drift between the current codebase and the map without modifying anything.

Architecture skill

An auto-activating skill that answers "how does this codebase work?" directly from your existing map data.

Markdown export

Every map is also written to docs/architecture-map.md — versionable, diffable, great for PR descriptions and onboarding docs.

Commands

CommandWhat it does
/archmapGenerate a full architecture map from scratch
/archmap <path>Map a specific subdirectory or project
/archmap --refreshRegenerate map for current project (full re-explore)
/archmap:repairDetect and surgically fix stale / broken maps
/archmap:repair --layoutFix only layout issues (overlaps, spacing)
/archmap:repair --detailsFix only missing module details
/archmap:focus <module>Deep-dive and repair a specific module
/archmap:diffShow architectural drift since last map (read-only)

Configuration

Drop an .archmap.json in your project root to override defaults. Every field is optional.

{
  "exclude": ["node_modules", "dist", ".git"],
  "tiers": { "src/api/": "api", "src/models/": "data" },
  "pinned": ["config-module"],
  "output": {
    "html": "docs/architecture.html",
    "markdown": "docs/architecture-map.md"
  },
  "theme": "dark",
  "hooks": { "sessionStart": false }
}

See the README for the full field reference and the list of tier keys.

FAQ

Does this send my code anywhere?

Archmap runs entirely inside Claude Code's agent dispatch — your code is read by the Explorer agent, structured into modules and edges, and written to a local HTML + markdown file. The generated output is self-contained (no external scripts, fonts, or CDN). Nothing is posted anywhere except through Claude Code itself.

Will it work on a huge monorepo?

The explorer agent caps module count (80) and honours .archmap.json exclude paths. For repos with thousands of files, narrow the scope with /archmap <path> or exclude build artifacts and vendored code up front. Hooks use git ls-files so they respect .gitignore for free.

What languages are supported?

Any — the explorer agent is language-agnostic. Built-in tier defaults favour common web, systems, and application-framework layouts. For specialised repos (compilers, data pipelines, infra-as-code) you can override tier assignments in .archmap.json.

How is this different from running a static analyser?

Static analysers produce call graphs; archmap produces architecture maps — human-legible groupings by role (entry / api / data / runtime / etc.), with narrative descriptions and notes per module. It's closer to the diagram a senior engineer draws on the whiteboard than a dependency graph dumped from a parser.

Can I edit the map by hand?

Yes. The data lives as JS variables inside the HTML (const modules = [...], const edges = [...]). Hand-edit, save, open in a browser. Just run /archmap:repair afterward if you want the plugin to also pick up your changes.

The SessionStart hook is chatty. Can I turn it off?

Set "hooks": { "sessionStart": false } in your .archmap.json.

GitHub Pages for my project already uses /docs — will archmap conflict?

Override output.html in .archmap.json to point anywhere you like — e.g. .archmap/map.html.

Can I contribute a new theme?

Yes, please. Open a PR adding a new key to the THEMES object in templates/archmap-template.html — see CONTRIBUTING.md.
